# Synthesis Patterns

Use this reference when combining two or more collaborator conversations.

## Cross-Conversation Output

```markdown
# 多方对话综合分析：{主题}

## 总体结论

{最重要的综合判断。}

## 已形成共识

- {共识 1}
- {共识 2}

## 明显分歧

- {分歧 1：谁和谁不同，差异在哪里}
- {分歧 2}

## 反复出现的主题

- {主题 1}
- {主题 2}

## 对 AI 原生平台的架构含义

### Business Layer

- {综合影响}

### Application Layer

- {综合影响}

### Technical Layer

- {综合影响}

### Data Layer

- {综合影响}

## 决策建议

- {建议进入决策会的问题}
- {建议暂缓的问题}
- {建议立即验证的问题}

## 需要继续确认的问题

- {问题 1}
- {问题 2}
```

## Synthesis Heuristics

- Treat repeated claims from different roles as stronger signals, but not automatically as truth.
- Highlight contradictions between executive ambition, product feasibility, technical constraints, and customer reality.
- Distinguish consensus from silence. A topic is not aligned just because nobody objected.
- Preserve minority views when they expose technical, market, or organizational risk.
- Convert vague excitement into testable hypotheses.
- Convert vague concern into concrete risk statements and validation paths.

## AI-Native Interpretation

For each cross-conversation synthesis, ask:

- Would the platform still exist without AI, only worse, or would the core product logic collapse?
- Which workflow becomes impossible or meaningfully different because of AI?
- Which proprietary data loop makes the platform better over time?
- Which decisions depend on model capability, data availability, or human trust?
- Which parts are truly AI-native versus AI-enhanced features attached to a conventional platform?
